## Bifunctors
The morphism in the category $\mathbf{Cat}$ are functors. Just like there are functions of 2 arguments there can be functors of 2 arguments, or a bifunctor.
Functorality mean that a bifunctor has to map morphism as well. In this case it must map a pair of morphisms, one from C and one from D, to a morphism in E
![[Pasted image 20250710182642.png]]
A pair of morphisms is just a single morphism in the product category CxD.

We define a morphism in a cartesian product of categories as a pair of morphisms which goes from one pair of objects to another pair of objects

An easier way to think about bifunctors would be to consider them functor in each argument separately. However, in general, separate functoriality is not enough to prove joint functoriality. Categories in which joint functoriality fails are called premonoidal.

## Product and Coproduct Bifunctors

```haskell
instance Bifunctor (,) where
    bimap f g (x, y) = (f x, g y)
```

An important instance of a Bifunctor is the categorical product.

bimap :: (a -> c) -> (b -> d) -> (a, b) -> (c, d)

```haskell
instance Bifunctor Either where
	bimap f _ (Left x) = Left (f x)
	bimap _ g (Right y) = Right (g y)
```


## Functorial Algebraic Data Types
We have seen that sums and products are functorial and we know functors compose so if we can show that the basic building blocks of ADTs are functorial then we can show that parametrized ADT are also functorial.

There are types that ignore its type parameter like Nothing in Maybe and Nil in List they are equivalent to the Const functor.

There are also types that just encapsulate the type itself, like Just in Maybe, these are equivalent to the identity functor.

```haskell
data Maybe a = Nothing | Just a
```
